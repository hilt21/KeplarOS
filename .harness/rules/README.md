# Harness Rules

Load all rule files in this directory before any implementation phase.

Start with:

- `.harness/rules/coding-discipline.md`

Rules define project-level safety boundaries. Agent and skill files define phase-specific workflows.

## Preferred orientation

Before deep-diving into prose specs, agents and humans should load the
token-lean codemaps at `docs/CODEMAPS/`. They cover the same domain in
~550–850 tokens each and are the fastest way to orient:

- `docs/CODEMAPS/README.md` — dispatch index (by role / by task)
- `docs/CODEMAPS/architecture.md` — 5 layers + key invariants
- `docs/CODEMAPS/backend.md` — REST + SSE surface
- `docs/CODEMAPS/data.md` — 11 tables + migrations
- `docs/CODEMAPS/frontend.md` — pages + components + stores
- `docs/CODEMAPS/dependencies.md` — what's installed vs declared

Codemaps are regenerated by the `update-codemaps` skill; the analysis
report is at `.reports/codemap-diff.txt`.

## 1. 发起人用户故事

### US-001: 创建目标空间

| 属性 | 内容 |
|------|------|
| **故事编号** | US-001 |
| **角色** | 发起人 |
| **故事描述** | 作为发起人，我希望在系统中创建一个目标空间，以便开始一个新的业务目标协作 |
| **触发条件** | 发起人点击"创建目标空间"按钮 |
| **前置条件** | 发起人已登录系统 |

**交互流程：**

```
发起人                    系统                      AI (Backlog Refiner)
  |                        |                            |
  |-- 输入自然语言目标 -->|                            |
  |                        |--- 解析目标 ------------->|
  |                        |<-- 返回 YAML Story ------|
  |                        |--- 创建 Goal Space ------>|
  |                        |<-- 确认创建 ---------------|
  |<-- 显示目标空间 Dashboard --|                       |
```

**正常路径：**
1. 发起人输入自然语言目标描述
2. 系统调用 Backlog Refiner 解析目标
3. 系统展示生成的 YAML Story 预览
4. 发起人确认或编辑 YAML Story
5. 系统创建目标空间并展示 Dashboard

**异常路径：**
- AI 解析失败 → 显示错误，允许发起人手动编辑
- 目标包含敏感词 → 提示发起人确认

### US-002: 启动运行会话

| 属性 | 内容 |
|------|------|
| **故事编号** | US-002 |
| **角色** | 发起人 |
| **故事描述** | 作为发起人，我希望启动一个运行会话，让系统自动开始拆解和执行任务 |
| **触发条件** | 发起人点击"启动会话"按钮 |
| **前置条件** | 存在已创建的目标空间 |

**交互流程：**

```
发起人                    系统                   AI 角色群                   卡片
  |                        |                        |                        |
  |-- 点击"启动会话" -->|                        |                        |
  |                        |--- 启动会话 --------->|                        |
  |                        |                        |-- 生成卡片 --------->|
  |                        |<-- 会话启动成功 -------|                        |
  |                        |                        |--- 分发到 Backlog -->|
  |<-- 显示会话状态面板 --|                        |                        |
```

**正常路径：**
1. 发起人点击"启动会话"
2. 系统创建新的会话上下文
3. Backlog Refiner 生成初始任务卡片
4. 卡片自动分发到 Todo Orchestrator
5. 系统通过 SSE 实时推送状态更新

**异常路径：**
- 会话启动失败 → 显示错误，允许重试
- AI 角色响应超时 → 卡片进入 Blocked 状态

### US-004: 人工确认关键任务

| 属性 | 内容 |
|------|------|
| **故事编号** | US-004 |
| **角色** | 发起人 |
| **故事描述** | 作为发起人，我希望对高风险任务进行人工确认，确保关键决策经过审批 |
| **触发条件** | 系统推送高风险任务待确认 |
| **前置条件** | 存在需要确认的高风险任务 |

**交互流程：**

```
发起人                    系统                   AI (Review Guard)
  |                        |                        |
  |<-- 推送待确认任务 --|                        |
  |                        |                        |
  |-- 审阅任务详情 -->|                        |
  |<-- 显示风险详情 ----|                        |
  |                        |                        |
  |-- 确认/拒绝 ------>|                        |
  |                        |--- 转发确认 --------->|
  |<-- 更新任务状态 ----|                        |
```

**决策选项：**
- 确认放行 → 任务进入下一阶段
- 拒绝并退回 → 任务退回给链路用户
- 要求补充信息 → 通知链路用户补充

## 4. 验收标准

### 4.1 发起人验收标准

#### AC-001: 创建目标空间

```
验收标准 1: 发起人输入自然语言目标后，30秒内系统返回 YAML Story
验收标准 2: YAML Story 包含 goal、constraints、acceptance_criteria、tasks 四个必要字段
验收标准 3: 发起人可编辑 YAML Story 后确认创建
验收标准 4: 目标空间创建后自动出现在发起人的 Dashboard 中
```

#### AC-002: 启动运行会话

```
验收标准 1: 点击启动后，系统在 5 秒内开始处理第一张卡片
验收标准 2: SSE 连接建立成功，实时看到状态更新
验收标准 3: 会话启动失败时，显示明确错误信息并允许重试
```

#### AC-003: 查看目标进度

```
验收标准 1: Dashboard 显示目标完成百分比
验收标准 2: Dashboard 显示活跃/阻塞/已完成卡片数量
验收标准 3: 高风险任务有红色标识
验收标准 4: 点击卡片可查看详情
```

#### AC-004: 人工确认

```
验收标准 1: 高风险任务出现在"待确认"列表
验收标准 2: 发起人可选择"确认"、"拒绝"、"补充信息"三种操作
验收标准 3: 操作后任务状态正确变更
验收标准 4: 操作记录出现在审计轨迹中
```

### 4.2 链路用户验收标准

#### AC-005: 认领任务卡片

```
验收标准 1: 待认领卡片显示在"我的卡片"列表
验收标准 2: 点击认领后，卡片状态变更为"Todo"
验收标准 3: 已认领的卡片不能再被其他人认领
```

#### AC-006: 补充业务上下文

```
验收标准 1: 卡片详情页显示"补充上下文"入口
验收标准 2: 保存后上下文内容在卡片详情中展示
验收标准 3: AI 基于新上下文重新分析
```

#### AC-007: 处理阻塞

```
验收标准 1: Blocked 卡片出现在"阻塞处理"列表
验收标准 2: 显示阻塞原因和处理建议
验收标准 3: 修复后卡片状态变更为"Todo"、"Dev" 或 "Review"，由阻塞解除目标决定
验收标准 4: 连续 3 次阻塞后触发人工接管通知
```

#### AC-008: 确认节点结果

```
验收标准 1: 节点内所有卡片完成后，显示"确认结果"按钮
验收标准 2: 确认前显示节点汇总预览
验收标准 3: 确认后节点状态变更为"Done"
```

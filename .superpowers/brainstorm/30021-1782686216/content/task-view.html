关键设计点（针对你提的 4 个关键词）

Persistent multi-pane shell

  • Goal Space 视图 / Task 视图共用同一组 3 栏 DOM 结构，pane 内容随 context 切换，shell 永 unmount
  • TopBar 跨 context 持久，只有 "Railway Metro / Main board / CARD-004" 这一行面包屑变
  • 动画用 pane-level opacity + translateY 8px（Codex spec），不是 page-level 路由切换

Master–detail navigation

  • Master（左）= 当前 board 的 task list，state dot + display_id + title（mono / ellipsis）
  • 选中 task 切 primary timeline + detail runtime（in-place，URL 同步为 /goal-spaces/[id]/tasks/[taskId] 但不 unmount shell）
  • Master 顶部 filter input 用于任务搜索

Timeline-based conversational UI

  • 5 种 message 类型：user / agent thinking / tool call / confirmation / agent response——每种有独立的左侧 icon + 配色
  • Streaming 时左侧 icon pulse 1.6s，工具完成后变实心
  • Human confirmation 卡片是 border-left 3px 警示色 + Approve/Reject 按钮（Codex spec 8.7 适配）
  • 底部 multi-line input 复用现有命令面板的 submit 模型

Progressive disclosure

  • Detail 默认只显示 CARD id / state / assignee / token——always-visible
  • Modified files / Plan / Token / Audit 用 accordion 折叠，hover 不展开（避免 hover-roulette），click 切换
  • 第一个 accordion（Modified files）默认展开——"当前 task 改了什么" 是 user 最高频需要的信息

CSS overflow 修复（直接示例在 master 列表）

/* 默认：单行省略 */
.task-title {
  flex: 1;
  min-width: 0;            /* 关键：允许 flex child shrink */
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
  transition: white-space 120ms ease-out, background 120ms ease-out;
  border-radius: 2px;
  padding: 0 2px;
}

/* hover：展开为多行 + 高亮 */
.task-title:hover,
.task-title:focus-within {
  white-space: normal;
  overflow: visible;
  background: var(--color-surface-elevated);
  z-index: 5;
  position: relative;
}

上面 mockup 的 master 列表（"CARD-001 Track geometry" 等）就是这套 CSS 的运行效果——窗口缩小时只省略 task title 部分，display_id 永远完整。
